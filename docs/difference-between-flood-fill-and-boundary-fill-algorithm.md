# 洪水填充和边界填充算法的区别

> 原文:[https://www . geesforgeks . org/洪水填充和边界填充算法之间的差异/](https://www.geeksforgeeks.org/difference-between-flood-fill-and-boundary-fill-algorithm/)

**[洪水填充算法](https://www.geeksforgeeks.org/flood-fill-algorithm-implement-fill-paint/) :**
洪水填充算法也被称为种子填充算法。它确定连接到多维数组中给定节点的区域。该算法的工作原理是在图像的内部填充或重新着色包含不同颜色的选定区域，从而填充或重新着色图像的边界。它通常用一幅画来说明，画中的一个街区以各种不同的颜色区域为边界。为了绘制这些区域，我们将替换特定的内部颜色，而不是发现边界颜色值。这是该方法的基本原理，因为洪水填充算法而被理解。
现在，有两种方法将用于通过连接像素来创建无限边界–4 连接和 8 连接方法。在 4 连接方法中，像素最多可以有四个相邻像素，分别位于当前像素的适当位置、左侧、上方和下方。相反，在 8 连接方法中，它可以有 8 个，并且对照 4 个对角像素检查相邻位置。所以，这两种方法中的任何一种通常都不会重画内点。

**[边界填充算法](https://www.geeksforgeeks.org/boundary-fill-algorithm/)** :
它遵循一种方法，其中区域填充从区域内部的某个程度开始，并朝着边界绘制内部。在边界包含单一颜色的情况下，填充算法在向外的方向上逐像素地继续，直到遇到边界颜色。边界填充算法通常主要在交互式绘画包中实现，其中内部点很容易选择。
边界填充的功能从接受室内点(x，y)的坐标开始，边界颜色和填充颜色成为输入。从(x，y)开始，该方法检查相邻位置，以确定它们是否是边界颜色的一部分。如果它们不是来自边界颜色，那么它们会被涂上填充颜色，并且它们的相邻像素会根据条件进行测试。当检查到世界边界颜色之前的所有像素时，该过程结束。

洪水填充和边界填充算法的区别:

| 洪水填充算法 | 边界填充算法 |
| --- | --- |
| 它可以处理包含不止一种边界颜色的图像。 | 它只能处理包含单一边界颜色的图像。 |
| 洪水填充算法比边界填充算法相对较慢。 | 边界填充算法比洪水填充算法更快。 |
| 在泛洪填充算法中，可以使用随机颜色来绘制内部部分，然后用新颜色替换旧颜色。 | 在边界填充算法中，通过连续搜索边界颜色来绘制内部点。 |
| 它需要大量的内存。 | 边界填充算法的内存消耗相对较低。 |
| 洪水填充算法简单有效。 | 边界填充算法的复杂度很高。 |
